## Your Mission

Build a JavaScript frontend that consumes a graph-based issue tracking API. The system manages **nodes** (issues, tasks, features, people) connected by **links** (relationships). Think: a simplified Jira with explicit relationship graphs.

## Error Handling

All API responses follow this pattern:

```
Success:                              Error:
â”€â”€â”€â”€â”€â”€â”€â”€â”€                             â”€â”€â”€â”€â”€â”€
{                                     {
  "success": true,                      "success": false,
  "node": { ... },                      "node": null,
  "message": ""                         "message": "Title is required"
}                                     }
```

**HTTP Status Codes:**
| Code | Meaning |
|------|---------|
| 200 | Success |
| 400 | Bad request (validation error) |
| 404 | Node/link not found |
| 500 | Server error |

**Handle These Errors:**
| Error Message | User-Facing Message |
|---------------|---------------------|
| "Title is required" | "Please enter a title" |
| "Unknown node type" | "Invalid issue type selected" |
| "not found" | "This issue no longer exists" |
| "Link already exists" | "This relationship already exists" |
| "Unknown link type" | "Invalid relationship type" |

```javascript
const API_BASE = '/api';

export const api = {

  nodes: {
    list: async () => {
      const res = await fetch(`${API_BASE}/nodes`);
      return res.json();
    },
    
    listByType: async (type) => {
      const res = await fetch(`${API_BASE}/nodes/type/${type}`);
      return res.json();

      const res = await fetch(`${API_BASE}/nodes`, {

      return res.json();
    },

      });
      return res.json();

  links: {
```

## Key Implementation Notes

1. **Labels are primary identifiers** - Use `Bug-27` in URLs, not node_id
2. **Type from label** - Parse `Bug-27` â†’ type is `bug` (lowercase prefix before hyphen)
3. **Bidirectional links** - Creating a link updates both nodes automatically
4. **Status options are type-specific** - Load from `/api/types/{name}` for dropdown options
5. **Link type constraints** - Some link types only work between certain node types
6. **Colors from type config** - Use `node_type.color` for consistent visual styling
